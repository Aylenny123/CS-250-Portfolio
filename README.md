# CS 250 Portfolio

## Sprint Review and Retrospective
This repository contains my final project for CS 250, which includes a Sprint Review and Retrospective based on Agile development practices.

## Reflection
How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?:
Throughout this course, I learned how to interpret user needs and turn them into working features by using user stories. User stories help break down what the user wants into smaller, clear requirements that are easier to understand and implement. Instead of trying to build everything at once, user stories allowed me to focus on one feature at a time while keeping the user’s perspective in mind. This made development more organized and helped ensure the final product matched user expectations.

How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?:
When approaching program development, I now rely more on Agile processes rather than a traditional waterfall approach. Agile allows for flexibility, continuous feedback, and the ability to adjust when requirements change. I learned the importance of working in small increments, testing early, and improving as I go. In the future, I plan to continue using practices like user stories, iterative development, and regular feedback because they help improve both the development process and the final product.

What does it mean to be a good team member in software development?:
Being a good team member in software development means communicating clearly, collaborating with others, and being open to feedback. Throughout this course, I saw how important it is to ask questions, share ideas, and support teammates when challenges come up. A strong team member also takes responsibility for their work while still contributing to the overall success of the team. Good teamwork leads to better results, fewer misunderstandings, and a more efficient development process.
